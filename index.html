Quick Start

New user
For New Users
Clone, run setup, and launch. No configuration needed.
Returning
For Troy
Clone engine + private user repo for full memory restoration.

New Users

  1. git clone https://github.com/Cothek/glitch-ai.git
  2. cd glitch-ai
  3. .\setup.bat — initializes engine + launches profile wizard
  4. .\launch-glitch.bat — start using Glitch

Your data stays local under user/ — nothing shared.

After setup, see First Run Only below to download required tools.

Troy (Returning User)

  1. git clone https://github.com/Cothek/glitch-ai.git
  2. cd glitch-ai
  3. git submodule update --init --recursive
  4. git clone https://cothek@github.com/Cothek/glitch-user-troy.git user/
  5. .\launch-glitch.bat

The cothek@ prefix tells Windows Credential Manager to auto-select your stored PAT — no login pop-up.

After cloning, see First Run Only below to download required tools.

GitHub login pop-up? If you get one after git submodule sync, the submodule remote got reset. Fix it:
cd glitch-memorycore
git remote set-url origin https://cothek@github.com/Cothek/glitch-engine.git

First Run Only — Everyone

Run .\scripts\bootstrap.ps1 to download OpenCode and optional tools (Handy voice input, Cloudflare Tunnel). This applies to everyone — new users and returning users alike. The script checks what's already present and skips anything downloaded previously, so subsequent runs are fast.

How It Works

Glitch is split into three independent layers:

Public
Engine
Cothek/glitch-engine — Core identity, 23+ skills, plugins, library. No user data.
Private
User Data
Cothek/glitch-user-troy — Personal memory, diary, decisions, projects. Troy only.
Public
Launcher
Cothek/glitch-ai — OpenCode binary, launch scripts, config, bootstrap.

Launch Flow

  1. launch-glitch.bat calls scripts/launch.ps1
  2. Detects your user profile (user/main-memory.md or user/{name}/main-memory.md)
  3. Generates a runtime opencode.json with engine + user instruction paths (string-level regex — never re-serializes JSON, preserves escaping)
  4. Starts OpenCode with the generated config
  5. On exit, restores the engine-only base config

Access from Anywhere

Start OpenCode as a web server proxied through Cloudflare Tunnel. No open ports, no VPN required.

Terminal 
.\scripts\setup-tunnel.ps1    # One-time: authenticate + create tunnel
.\serve-glitch.bat            # Each session: starts server + tunnel

Login with username opencode and the auto-generated password shown in the terminal. Configure your domain via .env:

# Copy .env.example to .env and set:
GLITCH_DOMAIN=glitch.yourdomain.com

Modifying Config / Launch Scripts

Changes to opencode.json, launch.ps1, serve-glitch.ps1, or any .bat launcher require:

  1. A review pass (@reviewer) before applying
  2. validate-config.ps1 passing after the change
  3. A restart of OpenCode to pick up the new config
R14 — Immutable Rule. This is enforced by glitch-memorycore/prompt-rules.md. Every config/launch change must pass review before it lands.

To validate the config at any time:

.\scripts\validate-config.ps1

Requirements

  • GitHub account (for cloning)
  • NVIDIA GPU with CUDA (recommended for Handy voice-to-text speed)
